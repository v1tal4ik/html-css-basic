# Структурна розмітка

## Що таке структурна розмітка?

Структурна розмітка — це використання семантичних HTML-тегів для визначення логічної структури сторінки.

Основні структурні теги:

- `<header>` — шапка сайту або секції.
- `<nav>` — навігація.
- `<main>` — основний контент.
- `<section>` — секція контенту.
- `<footer>` — підвал сайту або секції.

## Важливо

- Структурна розмітка покращує доступність та SEO.
- Кожна сторінка повинна мати чітку логічну структуру.
- Семантичні теги замінюють безликі `<div>`.

# Шапка сайту: тег header

## Що таке тег header

Тег `<header>` визначає шапку сайту або секції. Зазвичай містить логотип, навігацію та інші вступні елементи.

Приклад:

<header>
  <a href="/">
    <img src="./images/logo.svg" alt="Назва компанії" width="120" height="40" />
  </a>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
      <li><a href="#contacts">Contacts</a></li>
    </ul>
  </nav>
</header>

## Важливо

- `<header>` — це шапка всієї сторінки або окремої секції.
- На сторінці може бути кілька `<header>`, але зазвичай один головний.
- Не плутати з тегом `<head>` — це різні речі.

# Логотип

## Як верстати логотип

Логотип — це зображення-посилання, яке веде на головну сторінку.

Приклад:

<a href="/">
  <img src="./images/logo.svg" alt="Назва компанії" width="120" height="40" />
</a>

## Важливо

- Логотип завжди є посиланням на головну сторінку (`href="/"`).
- Атрибут `alt` містить назву компанії або сайту.
- Зазвичай розміщується першим елементом у `<header>`.

# Навігація: тег nav

## Що таке тег nav

Тег `<nav>` визначає блок навігаційних посилань.

Приклад:

<nav>
  <ul>
    <li><a href="#home">Home</a></li>
    <li><a href="#about">About</a></li>
    <li><a href="#services">Services</a></li>
    <li><a href="#contacts">Contacts</a></li>
  </ul>
</nav>

## Важливо

- `<nav>` використовується для основної навігації сторінки.
- Всередині — список `<ul>` з посиланнями.
- На сторінці може бути кілька `<nav>` (наприклад, у шапці та підвалі).

# Посилання-якір

## Що таке посилання-якір

Посилання-якір — це посилання, яке веде до певного місця на тій самій сторінці.

Приклад:

<a href="#about">Про нас</a>

<!-- Десь нижче на сторінці -->
<section id="about">
  <h2>Про нас</h2>
  <p>Текст розділу.</p>
</section>

## Важливо

- Значення `href` починається з `#` і відповідає `id` цільового елемента.
- При натисканні сторінка прокручується до елемента з відповідним `id`.
- Використовується для навігації в межах однієї сторінки.

# Тег main

## Що таке тег main

Тег `<main>` визначає основний контент сторінки, який є унікальним для цієї сторінки.

Приклад:

<main>
  <section id="hero">
    <h1>Ласкаво просимо</h1>
    <p>Головний банер сайту.</p>
  </section>

  <section id="about">
    <h2>Про нас</h2>
    <p>Інформація про компанію.</p>
  </section>
</main>

## Важливо

- На сторінці може бути лише один `<main>`.
- `<main>` не вкладається в `<header>`, `<footer>`, `<nav>`, `<aside>` або `<article>`.
- Містить контент, унікальний для конкретної сторінки.

# Підвал сайту. Тег footer

## Що таке тег footer

Тег `<footer>` визначає підвал сайту або секції. Зазвичай містить контактну інформацію, копірайт, посилання на соцмережі.

Приклад:

<footer>
  <address>
    <ul>
      <li>
        <a href="tel:+380681234567">+380 (68) 123-45-67</a>
      </li>
      <li>
        <a href="mailto:hello@example.com">hello@example.com</a>
      </li>
    </ul>
  </address>
  <p>&copy; 2024 Назва компанії. Усі права захищені.</p>
</footer>

## Важливо

- `<footer>` — підвал сторінки або секції.
- Зазвичай містить копірайт, контакти, посилання на соцмережі.
- На сторінці може бути кілька `<footer>` (для секцій), але один головний.

# Посилання на соцмережі

## Як оформити посилання на соцмережі

Посилання на соціальні мережі оформлюються як список у підвалі сайту.

Приклад:

<ul>
  <li>
    <a href="https://facebook.com" target="_blank" rel="noopener noreferrer">
      <img
        src="./images/facebook.svg"
        alt="Facebook"
        width="24"
        height="24"
      />
    </a>
  </li>
  <li>
    <a href="https://instagram.com" target="_blank" rel="noopener noreferrer">
      <img
        src="./images/instagram.svg"
        alt="Instagram"
        width="24"
        height="24"
      />
    </a>
  </li>
  <li>
    <a href="https://twitter.com" target="_blank" rel="noopener noreferrer">
      <img
        src="./images/twitter.svg"
        alt="Twitter"
        width="24"
        height="24"
      />
    </a>
  </li>
</ul>

## Важливо

- Посилання на соцмережі відкриваються в новій вкладці (`target="_blank"`).
- Обов'язково додаємо `rel="noopener noreferrer"` для безпеки.
- Кожне посилання — окремий `<li>` у списку.

# Символьні підстановки

## Що таке символьні підстановки

Символьні підстановки (HTML entities) — це спеціальні коди для відображення символів, які мають особливе значення в HTML.

Приклад:

&amp;copy; — ©
&amp;lt; — <
&amp;gt; — >
&amp;amp; — &
&amp;nbsp; — нерозривний пробіл
&amp;laquo; — «
&amp;raquo; — »

## Важливо

- Використовуються для символів, які неможливо ввести напряму.
- `&copy;` — знак копірайту, часто використовується у `<footer>`.
- `&lt;` та `&gt;` — для відображення кутових дужок як тексту.

# Символьні підстановки. Верстка символів коду

## Як верстати символи коду

Щоб відобразити HTML-теги як текст на сторінці, потрібно замінити кутові дужки на символьні підстановки.

Приклад:

<p>Тег &amp;lt;a&amp;gt; створює посилання.</p>
<p>Тег &amp;lt;img /&amp;gt; вставляє зображення.</p>

Результат на сторінці:

Тег <a> створює посилання.
Тег <img /> вставляє зображення.

## Важливо

- `&lt;` замінює `<`, а `&gt;` замінює `>`.
- Без підстановок браузер інтерпретує теги як HTML-код.
- Використовується при написанні документації або навчальних матеріалів.

# Секції контенту тег section

## Що таке тег section

Тег `<section>` визначає тематичну групу контенту, зазвичай з заголовком.

Приклад:

<main>
  <section id="hero">
    <h1>Головний заголовок</h1>
    <p>Вступний текст сторінки.</p>
    <button type="button">Дізнатись більше</button>
  </section>

  <section id="about">
    <h2>Про нас</h2>
    <p>Інформація про компанію.</p>
  </section>

  <section id="services">
    <h2>Наші послуги</h2>
    <ul>
      <li>Веб-дизайн</li>
      <li>Розробка</li>
      <li>Підтримка</li>
    </ul>
  </section>
</main>

## Важливо

- Кожна `<section>` повинна мати заголовок (`<h1>`–`<h6>`).
- Використовується для логічного поділу контенту на блоки.
- Не використовувати `<section>` як заміну `<div>` для стилізації.

# Висновки

- Структурна розмітка визначає логічну структуру сторінки.
- `<header>` — шапка сайту з логотипом та навігацією.
- Логотип — зображення-посилання на головну сторінку.
- `<nav>` — блок навігаційних посилань.
- Посилання-якір веде до елемента з відповідним `id` на тій же сторінці.
- `<main>` — основний унікальний контент сторінки (один на сторінку).
- `<footer>` — підвал з контактами, копірайтом, соцмережами.
- Посилання на соцмережі відкриваються в новій вкладці з `rel="noopener noreferrer"`.
- Символьні підстановки дозволяють відображати спецсимволи як текст.
- `<section>` — тематична група контенту із заголовком.
