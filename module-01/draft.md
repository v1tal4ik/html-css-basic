# Посилання

## Аналіз контенту

Перед початком верстки необхідно проаналізувати контент сторінки.

Це допомагає зрозуміти:

- Які теги використовувати.
- Яка структура документа.
- Де розміщувати посилання, списки, зображення.

## Важливо

- Спочатку аналізуємо макет або текст.
- Визначаємо логічні блоки контенту.
- Підбираємо відповідні HTML-теги для кожного блоку.

# Блокові та рядкові елементи

## Що таке блокові елементи

Блокові елементи займають усю доступну ширину і починаються з нового рядка.

Приклад блокових елементів:

<div>Блок</div>
<section>Секція</section>
<p>Абзац</p>
<h1>Заголовок</h1>
<ul>Список</ul>

## Що таке рядкові елементи

Рядкові елементи займають лише стільки місця, скільки потрібно їх вмісту.

Приклад рядкових елементів:

<a href="#">Посилання</a>
<img src="https://placehold.co/50x50/green/white?text=OK" alt="" />
<span>Текст</span>
<button type="button">Кнопка</button>

## Важливо

- Блокові елементи можуть містити інші блокові та рядкові елементи.
- Рядкові елементи не повинні містити блокові елементи.
- Це базове правило вкладеності HTML.

# Список посилань

## Що таке список посилань

Навігаційне меню — це список посилань, обгорнутий у тег `<nav>`.

Приклад:

<nav>
  <ul>
    <li><a href="https://www.wikipedia.org">Wikipedia</a></li>
    <li><a href="https://developer.mozilla.org">MDN</a></li>
    <li><a href="https://www.google.com">Google</a></li>
  </ul>
</nav>

## Важливо

- Меню навігації завжди оформлюється як список.
- Тег `<nav>` вказує, що це навігаційний блок.
- Кожне посилання знаходиться в окремому `<li>`.

# Вкладеність тегів. Зображення-посилання

## Що таке зображення-посилання

Зображення можна обгорнути тегом `<a>`, щоб зробити його клікабельним.

Приклад:

<a href="https://www.wikipedia.org">
  <img
    src="https://picsum.photos/200/100"
    alt="Перейти на сайт"
    width="200"
    height="100"
  />
</a>

## Важливо

- Тег `<img>` вкладається всередину тегу `<a>`.
- Атрибут `alt` описує, куди веде посилання.
- Користувач натискає на зображення, щоб перейти за посиланням.

# Зображення-посилання в списку

## Як оформити зображення-посилання в списку

Часто іконки соціальних мереж оформлюються як список зображень-посилань.

Приклад:

<ul>
  <li>
    <a href="https://facebook.com" target="_blank" rel="noopener noreferrer">
      <img
        src="https://cdn.jsdelivr.net/npm/simple-icons@v9/icons/facebook.svg"
        alt="Facebook"
        width="24"
        height="24"
      />
    </a>
  </li>
  <li>
    <a href="https://instagram.com" target="_blank" rel="noopener noreferrer">
      <img
        src="https://cdn.jsdelivr.net/npm/simple-icons@v9/icons/instagram.svg"
        alt="Instagram"
        width="24"
        height="24"
      />
    </a>
  </li>
  <li>
    <a href="https://twitter.com" target="_blank" rel="noopener noreferrer">
      <img
        src="https://cdn.jsdelivr.net/npm/simple-icons@v9/icons/twitter.svg"
        alt="Twitter"
        width="24"
        height="24"
      />
    </a>
  </li>
</ul>

## Важливо

- Кожне зображення-посилання знаходиться в окремому `<li>`.
- Для зовнішніх посилань додаємо `target="_blank"` та `rel="noopener noreferrer"`.
- Атрибут `alt` вказує назву соціальної мережі.

# Вкладеність тегів. Основи

## Правила вкладеності

Теги можна вкладати один в одний, але є певні правила.

Правильно:

<ul>
  <li>
    <a href="#">
      <img src="https://picsum.photos/50/50" alt="" width="50" height="50" />
    </a>
  </li>
</ul>

Неправильно:

<a href="#">
  <a href="#">Посилання всередині посилання</a>
</a>

<p>
  <div>Блоковий елемент всередині рядкового</div>
</p>

## Важливо

- Не можна вкладати `<a>` в `<a>`.
- Не можна вкладати блокові елементи в `<p>`.
- Тег `<li>` може бути лише всередині `<ul>` або `<ol>`.

# Вкладеність: як визначити, що куди можна вкладати

## Правила визначення вкладеності

Щоб зрозуміти, що куди можна вкладати, використовуємо правила:

1. Блокові елементи можуть містити блокові та рядкові.
2. Рядкові елементи можуть містити лише рядкові.
3. `<p>` може містити лише рядкові елементи.
4. `<ul>` та `<ol>` можуть містити лише `<li>`.
5. `<a>` не може містити інший `<a>`.

## Важливо

- Якщо не впевнені — перевіряйте через валідатор Can I include.
- Браузер може відобразити невалідний код, але це не означає, що він правильний.
- Правильна вкладеність важлива для доступності та SEO.

## Де шукати інформацію про вкладеність

Корисні ресурси:

- Специфікація HTML Living Standard — стандарти та правила мови HTML.
- Довідник HTML-тегів — шпаргалка по тегах.
- Can I include (https://caninclude.glitch.me/) — швидка перевірка вкладеності.

Алгоритм перевірки через Can I include:

1. Вводимо тег, який хочемо вкласти (наприклад, li), у поле Child tag name.
2. Вводимо тег, куди вкладаємо (наприклад, ul), у поле Parent tag name.
3. Дивимося результат:
   - зелене — можна вкласти,
   - червоне — не можна,
   - жовте — можна з обмеженнями (наприклад, у `<a>` можна вкладати все крім інтерактивних елементів).

# Оформлення коду: відступи між розділами

## Правила відступів

Між логічними розділами сторінки додаються порожні рядки для кращої читабельності.

Приклад:

<header>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <section>
    <h1>Заголовок</h1>
    <p>Текст абзацу.</p>
  </section>

  <section>
    <h2>Інший розділ</h2>
    <p>Ще один абзац.</p>
  </section>
</main>

<footer>
  <p>© 2024</p>
</footer>

## Важливо

- Порожній рядок між `<header>`, `<main>`, `<footer>`.
- Порожній рядок між секціями всередині `<main>`.
- Відступ у 2 пробіли для кожного рівня вкладеності.

Давайте порівняємо на прикладі файлів:

- examples/bad-spacing.html — без порожніх рядків між розділами, код зливається в одну суцільну стіну.
- examples/good-spacing.html — порожні рядки візуально розділяють логічні блоки, код легко читати.

Також порівняйте семантичну та не семантичну розмітку:

- examples/non-semantic.html — все на div, без семантики.
- examples/semantic.html — правильні семантичні теги.

# Тег address

## Що таке тег address

Тег `<address>` використовується для контактної інформації.

Приклад:

<address>
  <ul>
    <li>
      <a href="tel:+380681234567">+380 (68) 123-45-67</a>
    </li>
    <li>
      <a href="mailto:hello@example.com">hello@example.com</a>
    </li>
  </ul>
</address>

## Важливо

- Використовується для контактних даних автора або організації.
- Зазвичай розміщується у `<footer>`.
- Може містити посилання на телефон, email, адресу.

# Посилання: значення атрибуту href tel та mailto

## Що таке tel та mailto

Спеціальні значення `href` дозволяють створювати посилання на телефон та email.

Приклад посилання на телефон:

<a href="tel:+380681234567">+380 (68) 123-45-67</a>

Приклад посилання на email:

<a href="mailto:hello@example.com">hello@example.com</a>

## Важливо

- `tel:` — відкриває додаток для дзвінків на мобільних.
- `mailto:` — відкриває поштовий клієнт.
- Номер телефону у `href` записується без пробілів та дужок.

# Посилання: атрибут download

## Що таке атрибут download

Атрибут `download` вказує браузеру завантажити файл замість переходу за посиланням.

Приклад:

<a href="./examples/price-list.txt" download>
  Завантажити прайс-лист
</a>

<a href="./examples/menu.txt" download="menu-2024">
  Завантажити меню
</a>

## Важливо

- Браузер завантажить файл, а не відкриє його.
- Можна вказати бажане ім'я файлу: `download="назва"`.
- Працює лише для файлів з того ж домену або з CORS-дозволом.

# Тег button

## Що таке тег button

Тег `<button>` створює кнопку на сторінці.

Приклад:

<button type="button">Натисни мене</button>

<button type="button">Buy now</button>

<button type="button">Leave a review</button>

## Різниця між button та a

- `<a>` — перенаправляє користувача на іншу сторінку або ресурс.
- `<button>` — виконує дію на поточній сторінці (відкрити модалку, надіслати форму тощо).

## Важливо

- Атрибут `type="button"` запобігає відправці форми за замовчуванням.
- Кнопка не є посиланням — вона виконує дію.
- Якщо потрібно перейти кудись — використовуємо `<a>`, якщо виконати дію — `<button>`.

# Висновки

- Перед версткою аналізуємо контент сторінки.
- Блокові елементи займають всю ширину, рядкові — лише потрібну.
- Навігація оформлюється як список посилань у `<nav>`.
- Зображення можна зробити посиланням, обгорнувши в `<a>`.
- Теги мають правила вкладеності, які не можна порушувати.
- Між розділами додаємо порожні рядки для читабельності.
- `<address>` — для контактної інформації.
- `tel:` та `mailto:` — спеціальні значення для телефону та email.
- Атрибут `download` змушує браузер завантажити файл.
- `<button>` виконує дію, `<a>` перенаправляє на іншу сторінку.
