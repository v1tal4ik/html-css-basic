# Алгоритм створення сторінки

## Процес створення сайту

Процес створення сайту складається з кількох етапів:

1. Аналіз макета — визначаємо структуру, секції, елементи.
2. Створення файлової структури — папки для зображень, іконок, стилів.
3. Написання HTML-розмітки — від загального до деталей.
4. Експорт зображень з макета — зберігаємо у відповідні папки.
5. Валідація коду — перевіряємо на помилки.

## Важливо

- Спочатку аналізуємо макет повністю, потім починаємо верстку.
- Працюємо поетапно: спочатку структура, потім деталі.
- Після кожного етапу перевіряємо результат у браузері.

# Базовий алгоритм створення HTML-розмітки

## Крок 1: Базова структура документа

Створюємо HTML-документ з необхідними мета-тегами.

Favicon — це маленька іконка сайту, яка відображається у вкладці браузера поруч з назвою сторінки. Підключається через тег `<link>` у `<head>`:

<link rel="icon" href="images/favicon.svg" type="image/svg+xml" />

- Рекомендований формат — SVG (векторний, чіткий на будь-якому екрані).
- Атрибут `rel="icon"` (не `"shortcut icon"` — це застарілий синтаксис).
- Для SVG вказуємо `type="image/svg+xml"`.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta
      name="description"
      content="Treat yourself or a loved one to our finest ingredients for a moment of pure delight!"
    />
    <meta name="keywords" content="Chocolate, milk, cacao" />
    <title>SIMPLY CHOCOLATE</title>
    <link rel="shortcut icon" href="images/favicon.svg" type="image/svg" />
  </head>
  <body></body>
</html>

## Крок 2: Експорт зображень та файлова структура

Перед версткою потрібно експортувати всі зображення з макета у відповідні папки.

## Коли який формат використовувати

SVG — іконки, логотипи, проста графіка (векторні зображення, не втрачають якість при масштабуванні):

- logo.svg — логотип
- icon-facebook.svg, icon-instagram.svg, icon-twitter.svg — іконки соцмереж
- arrow-down.svg — іконка стрілки
- favicon.svg — іконка вкладки

PNG — зображення з прозорим фоном (вирізані об'єкти без фону):

- ingridients/img-1.png — інгредієнти на прозорому фоні
- reviews/img-1.png — аватарки користувачів (круглі, без фону)

JPEG (JPG) — фотографії з суцільним фоном (не потребують прозорості, менший розмір файлу):

- products/img-1.jpg — фото продуктів (шоколадки на фоні)
- bestsellers/img-1.jpg — фото бестселерів
- made/image.jpg — фото процесу виробництва

## Важливо

- SVG — для всього, що можна намалювати вектором (іконки, лого).
- PNG — коли потрібна прозорість фону.
- JPEG — коли це фото без прозорості (менший розмір файлу).
- Називаємо файли латиницею, без пробілів, у нижньому регістрі.

## Крок 3: Абсолютні та відносні шляхи

Абсолютний шлях — повна адреса ресурсу від кореня:

https://facebook.com
https://cdn.jsdelivr.net/npm/simple-icons@v9/icons/facebook.svg

Відносний шлях — адреса відносно поточного файлу:

./images/icons/logo.svg
./images/products/img-1.jpg
./index.html

- `./` — поточна папка.
- `../` — папка на рівень вище.
- Для локальних ресурсів (зображення, стилі) використовуємо відносні шляхи.
- Для зовнішніх ресурсів (соцмережі, CDN) використовуємо абсолютні шляхи.

## Крок 4: Визначаємо основні структурні блоки

Додаємо header, main та footer — кістяк сторінки.

<body>
  <header id="home"></header>

  <main></main>

  <footer></footer>
</body>

## Оформлення коду: відступи між розділами

- Порожній рядок між `<header>`, `<main>`, `<footer>`.
- Порожній рядок між секціями всередині `<main>`.
- Відступ у 2 пробіли для кожного рівня вкладеності.
- Порожні рядки візуально розділяють логічні блоки і роблять код легким для читання.

# Валідація

## Що таке валідація HTML

Валідація — це перевірка HTML-коду на відповідність стандартам W3C.

Валідатор: https://validator.w3.org/nu/

## Як використовувати валідатор

1. Відкриваємо https://validator.w3.org/nu/.
2. Обираємо спосіб перевірки: URL, завантаження файлу або вставка коду.
3. Натискаємо "Check".
4. Аналізуємо помилки (errors) та попередження (warnings).
5. Виправляємо помилки у коді.

## Типові помилки валідатора та як їх виправити

Помилка: Bad value for attribute "href" on element "a": Must be non-empty.
Причина: атрибут `href` порожній — `<a href="">`.
Виправлення: вказати адресу або якір, наприклад `href="#"` або `href="#section-id"`.

Помилка: Element "h2" must not be empty.
Причина: тег `<h2></h2>` не має текстового вмісту.
Виправлення: додати текст заголовка або використати атрибут `hidden`, якщо заголовок візуально прихований — `<h2 hidden>Назва секції</h2>`.

Помилка: Bad value "image/svg" for attribute "type" on element "link".
Причина: неправильний MIME-тип для SVG.
Виправлення: замінити `type="image/svg"` на `type="image/svg+xml"`.

Помилка: The "shortcut" keyword is non-conforming.
Причина: `rel="shortcut icon"` — застарілий синтаксис.
Виправлення: замінити на `rel="icon"`.

Помилка: Named character reference was not terminated by a semicolon.
Причина: символ `&` не екранований — наприклад `Apple&Cranberry`.
Виправлення: замінити `&` на `&amp;` — `Apple&amp;Cranberry`.

## Виправлена верстка (фрагменти)

Було:

<link rel="shortcut icon" href="images/favicon.svg" type="image/svg" />

Стало:

<link rel="icon" href="images/favicon.svg" type="image/svg+xml" />

Було:

<a href="">
  scroll down
</a>

Стало:

<a href="#made">
  scroll down
</a>

Було:

<h2></h2>

Стало:

<h2>It all adds up to one exceptional taste sensation</h2>

Було:

<h3>Apple&Cranberry</h3>
<h3>Lime&Sea salt</h3>

Стало:

<h3>Apple&amp;Cranberry</h3>
<h3>Lime&amp;Sea salt</h3>

## Важливо

- Валідний код — обов'язкова вимога до верстки.
- Помилки (errors) потрібно виправляти обов'язково.
- Попередження (warnings) бажано враховувати.
- Валідація допомагає знайти помилки вкладеності, незакриті теги, відсутні атрибути.
- Після виправлень — повторно перевіряємо код у валідаторі.

# Висновки

- Процес створення сайту йде від аналізу макета до валідації.
- Базовий алгоритм: структура документа → основні блоки → наповнення секцій.
- Зображення експортуємо у відповідні папки з правильними форматами.
- Відносні шляхи — для локальних ресурсів, абсолютні — для зовнішніх.
- Валідація через https://validator.w3.org/nu/ — обов'язковий етап.
